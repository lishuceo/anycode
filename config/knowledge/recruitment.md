# TapTap Maker 团队招聘标准

## 团队定位

精英化、AI 化的产品研发团队。团队成员需要真正将 AI 作为核心生产力工具，而非辅助手段。

---

## 在招岗位

**1. Agent 研发工程师**
- 负责 AI Agent 系统的设计与开发，包括多 Agent 协作、任务编排、工具调用等核心能力
- 加分项：多 Agent 系统研究经验、Agent eval/benchmark 经验、游戏开发 Agent、美术生成 Agent 等垂直场景经验

**2. TA — PCG 方向**
- 负责程序化内容生成（Procedural Content Generation）相关的技术美术工作，衔接美术表现与引擎能力

**3. 引擎开发工程师**
- 参与 UrhoX 游戏引擎的开发，包括渲染、编辑器、跨平台等方向

**4. Post-Training 算法研究员/工程师**
- 负责开源模型在 AI 游戏创作平台（Maker + UrhoX）垂直场景的微调与优化
- 偏年轻高潜、性价比优先，实习生也可以

---

## 候选人通用要求

**必要条件（所有岗位适用）：**
- **AI 工具深度使用者** — 日常工作中重度使用 Coding Agent（Cursor / Claude Code 等），能举出具体的提效场景和量化结果，而非仅"了解"或"试过"
- **快速学习能力** — 能主动发现和掌握新 AI 工具，持续优化自己的工作流
- **有辨识度的个人成果** — 能展示出独特的价值（如：有影响力的开源贡献、独立完成过有挑战性的项目、发表过有深度的技术内容），而非仅有"大厂 + 常规业务"的经历

**画像要求（满足其一即可）：**
- **高潜力型**：年轻（≤5 年经验）、学习速度快、有饥饿感、聪明
- **成果突出型**：有行业公认的突出成果或深厚积累，经验年限不限

---

## 简历分析要求

分析候选人简历时，需重点验证以下内容：

**开源贡献核实：**
- 候选人提到的开源项目，必须去 GitHub 核实实际贡献大小
- 查看 PR 数量、代码量、是否为核心贡献者，排除只提了 1-2 个小 PR 就写"参与 XX 开源项目"的情况
- 关注贡献质量：是核心功能开发、架构设计，还是文档修改、typo 修复

**技能含金量判断：**
- 区分"使用了某个开源库/API"和"有自己的技术深度和创造性"
- 警惕简历中堆砌框架名称但缺乏深度的情况（如"熟悉 LangChain + LlamaIndex + AutoGen"但没有具体成果）
- 重点看候选人在项目中解决了什么难题、做了什么别人没做过的事
